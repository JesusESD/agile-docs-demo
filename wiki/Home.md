# Helios Team Wiki

The Wiki is for **lightweight, fast-changing reference content** that doesn't need the pull-request review workflow — a glossary, team norms, onboarding checklists, and how-to notes.

> **Wiki vs. the docs site — when to use which?**
>
> - Use the **docs site** (`/docs`) for anything that should be **reviewed and versioned with sign-off**: epics, stories, BRDs, release notes, reports.
> - Use the **Wiki** for quick reference the whole team can edit freely: glossary, conventions, tips.
