# Glossary

Shared terminology for the Helios team. Keep entries short and add freely.

| Term | Definition |
|------|-----------|
| **Acceptance Criteria** | The conditions a story must satisfy to be considered done, usually written Given/When/Then. |
| **BRD** | Business Requirements Document — the agreed business problem, objectives, and requirements. |
| **Definition of Done (DoD)** | The shared checklist every story/epic must meet before it's "done." |
| **Epic** | A large body of work delivering a capability, broken into stories. |
| **Idempotency key** | A unique token that ensures a repeated API request doesn't perform the action twice. |
| **Instant Payment** | An account-to-account transfer that settles in under 10 seconds, 24/7. |
| **PI Planning** | Program Increment planning — aligning objectives across multiple sprints. |
| **Story Points** | A relative estimate of effort/complexity for a user story. |
| **Velocity** | The average number of story points a team completes per sprint. |
